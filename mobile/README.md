# Solarsearch mobile (Capacitor iOS shell)

Wraps the existing web app (`tech.html`) in a native iOS shell so it can go to
TestFlight / the App Store, with native camera, geolocation and push. See
`docs/MOBILE_RELEASE.md` for the release plan and distribution decision.

> The `ios/` Xcode project is generated by `cap add ios` and **must be run on a Mac
> with Xcode** (or a CI builder like Codemagic / Xcode Cloud). It is not committed here.

## Build steps (on a Mac)
```bash
cd mobile
npm install
npm run copy:web        # copy tech.html -> www/index.html (+ src/lib/supabase.js)
npx cap add ios         # generates the ios/ Xcode project (Mac only)
npm run sync            # copy web + native plugins into the project
npx cap open ios        # open in Xcode -> sign with your team -> Product > Archive
```
Then in Xcode: set the Team (Apple Developer org account), a unique bundle id
(`net.solarme.solarsearch.tech`), app icons, and Archive → distribute to TestFlight.

Iterating on the web app? Re-run `npm run copy:web && npm run sync`. For fast local
iteration you can instead set `server.url` in `capacitor.config.ts` to the hosted app.

## Required — iOS permission strings (Info.plist)
`cap add ios` creates `ios/App/App/Info.plist`. Add these (or the app crashes / is
rejected) — the wording is shown to the user in the OS permission prompt:
```xml
<key>NSCameraUsageDescription</key>
<string>Take site photos for your solar assessment.</string>
<key>NSLocationWhenInUseUsageDescription</key>
<string>Tag site photos and confirm on-site attendance.</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>Attach drone or existing photos to a job.</string>
<key>NSPhotoLibraryAddUsageDescription</key>
<string>Save captured job photos.</string>
```

## App Store Connect checklist (see docs/MOBILE_RELEASE.md)
- Apple Developer **Organization** account (D-U-N-S number).
- **Privacy policy URL** + **App Privacy labels** (location, photos, identifiers).
- **In-app account deletion** (Guideline 5.1.1(v)) — add a delete-account action in the app.
- **Sign in with Apple** only if other social logins are offered (email/OTP alone doesn't need it).
- Demo/reviewer login if distributing publicly.

## Native plugin wiring (follow-up, not required for a first WebView build)
The web app already uses `navigator.geolocation`, `<input capture>` and `crypto.subtle`,
which work inside WKWebView. Upgrade to native plugins for reliability + push:
- `@capacitor/camera` — cleaner capture + control of the bytes we hash.
- `@capacitor/geolocation` — native permission + accuracy.
- `@capacitor/push-notifications` — APNs for job offers / ETA (needs server APNs).
